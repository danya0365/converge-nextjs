### Reporting Bugs

1. Check if the bug has already been reported in [Issues](https://github.com/danya0365/converge-nextjs/issues)
2. If not, create a new issue using the **Bug Report** template
3. Include as much detail as possible

### Suggesting Features

1. Check existing [Issues](https://github.com/danya0365/converge-nextjs/issues) for similar suggestions
2. Create a new issue using the **Feature Request** template
3. Describe the feature and why it would be useful

### Pull Requests

1. **Fork** the repository
2. **Clone** your fork
   ```bash
   git clone https://github.com/YOUR_USERNAME/converge-nextjs.git
   ```
3. **Create a branch** for your feature
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** following our coding standards
5. **Test** your changes
   ```bash
   yarn type-check
   yarn lint
   ```
6. **Commit** with a descriptive message
   ```bash
   git commit -m "feat: add new feature description"
   ```
7. **Push** to your fork
   ```bash
   git push origin feature/your-feature-name
   ```
8. **Open a Pull Request** against the `main` branch

## ğŸ“ Coding Standards

### TypeScript
- Use TypeScript for all new code
- Define proper types/interfaces
- Avoid `any` type

### React Components
- Use functional components with hooks
- Follow Clean Architecture patterns
- Keep components small and focused

### Styling
- Use Tailwind CSS classes
- Follow existing design patterns
- Support dark/light themes

### Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `style:` Code style changes (formatting, etc.)
- `refactor:` Code refactoring
- `test:` Adding or modifying tests
- `chore:` Maintenance tasks
